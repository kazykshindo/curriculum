◆基本用語

■ ネットワーク層
・IP
　：Interenet Protocol。相手先のアドレス情報をもとにパケットを送信する機能。インターネット層のプロトコル。
・IPヘッダ
　：パケットの先頭にある制御情報。送信元・宛先IPアドレスなどの通信制御用のデータが記載される。
・ICMP
　：Internet Control Message Protocol。IP通信の疎通状態を調査し、正常でない場合送信元へエラー情報を返したりする。通信を補佐するプロトコル。
・ICMPメッセージ
　：短いメッセージを送受信して相手方に問い合わせや通知を行うことができる。エコー要求やエコー応答など。
・IPアドレス
　：Internet Protocol Address。インターネットなどのネットワークに接続された通信機器の一台ごとに割り当てられた識別番号。IPv4とIPv6の2種類ある。
　・オクテット
　　：IPv4において8ビットずつカンマで区切られた部分を指す
　・ネットワーク部
　　：ネットワークを識別するアドレス。ネットワークに割り当てられたIPアドレス群の中でネットワーク自信を指し示すアドレス。
　・ホスト部
　　：コンピュータや核危機を識別するアドレス。ネットワーク内の個々のホストに割り当てられるアドレス。
　・クラス
　　：ネットワーク部とホスト部の境界をIPv4アドレスの上位数ビットによって決定する方式。
　・プライベートIPアドレス
　　：インターネットのようなグローバルなものではなく社内LANなど小規模なネットワークで使用されるIPアドレス。インターネット上では使うことができずプライベートネットワーク内でのみ使われる。
　・グローバルIPアドレス
　　：インターネットプロバイダに割り振られるIPアドレス。インターネットに接続するパソコンなどの機器に割り振られインターネット上で通信を行う際に必ず必要なIPアドレスである。
・ネットワークアドレス
　：ホスト部の2進数のビットがすべて0のアドレス。ネットワーク全体を指す。
・ブロードキャストアドレス
　：ホスト部の2進数のビットがすべて1のアドレス。あるネットワークに接続されているすべてのホストに送信するためのアドレス。
・ループバックアドレス
　：自分自身を示す仮想的なIPアドレス。ネットワークアドレスとして使用禁止とされている「127.0.0.0/8」のネットワーク
・サブネット
　：ブロードキャストを行う際にホスト数を絞りブロードキャストドメインを狭めるための技術。
・サブネットマスク
　：IPアドレスのネットワーク部とホスト部の境界を示す。32ビットで構成される。
　　・プレフィックス表記
　　　：何ビット目までがネットワーク部下を簡易的に示した表記のこと。IPアドレスの記載に続いて、「/サブネットマスク1の個数」で表記する。
・デフォルトゲートウェイ
・IPv6
　：IPアドレスを128bitのデータとして表現する。最大約340澗台のPCがインターネットに直接接続できる。
　・サブネットプレフィックス
　　：IPv4におけるインターネット部に相当。基本的に128bit中前半の64bitが該当する
　・インターフェイスID
　　：IPv4におけるホスト部に相当。通常のIPv6のノードはMACアドレスより算出する。
・ルータ
　・ルーティングテーブル
　　：ルータに記録される経路情報。ルーティング処理を行う際に参照にする。
　　・ネクストホップ
　　　：通信経路で次に転送する最も近いルータやインターフェイスのこと。
　・ARPテーブル
　　：IPアドレスと紐づけられたMACアドレスが保存されている
　　・ARP
　　　：MACアドレスを宛先のIPアドレスから求めるためのプロトコル。宛先MACアドレスが分からないとき、ARPリクエストをブロードキャストで送信する
　・コアルータ
　　：基幹ネットワークにおいてメインで使用される。通信キャリアやISPネットワークの通信を中継するために利用され大量のデータを転送する必要があるので高速なデータ転送が可能。
　・センタールータ
　　：企業で中心的に利用されるルータ。拠点となる本社やデータセンター等の企業内で中心となる場所。接続するためのネットワークを構成する。
　・エッジルータ
　　：企業で利用されるルータ。中心となる拠点を除く営業所や支店等の拠点間を接続するネットワークを構成する。
　・ブロードバンドルータ
　　：家庭環境やSOHO環境等でいんたーネット回線に接続することを目的に利用されるルータ。
・ルーティング
　：異なるネットワークにパケットを送信する際に最適な経路を決めること。
　・直接接続
　　：ルータが直接接続しているネットワークのルートのこと。
　・スタティックルーティング
　　：管理者が宛先ネットワークへの最適なルートを手動で設定したルート。
　・ダイナミックルーティング
　　：ルーティングプロトコルにより動的に追加されるルート。
　　・ディスタンスベクタ型
　　　：ホップ数（宛先到達までのデバイス数）をメトリックとする。自身のルーティングテーブルに隣接ルータから転送された情報をそのまま追加していく方法でルート情報のやり取りを行う。
　　・リンクステート型
　　　：各ルータが全体の接続情報を見て最適経路を選択する方式。ルータ同士でLSA（Link State Advertisement）という自信のネットワーク情報を交換しそれを用いてネットワーク全体を把握している。ルータに負担はかかるがルート再構成にかかる時間は短い、大規模ネットワーク向き。
　　・ハイブリッド型
　　　：ディスタンスベクタ型をベースに、リンクステート型を組み合わせた方式Cisco独自に拡張した方式のため一つでも非Cisco社製デバイスがあると成立しない。
■ トランスポート層
・コネクション型通信
　：通信を確立するもの同士での連絡を取り合って制御を行う方式。通信を確立するためにお互いが連絡を取り合うため信頼性があるといえる。
　・ACK
　　：Acknowledgementフラグ。コネクションを確立するときに最初に送信する。
・コネクションレス型通信
　：連絡を取り合っての制御はせず、相手の準備ができているかに関わらず送信するので信頼性がないといわれる。その分ネットワーク負荷が軽いため通信効率がいいメリットがある。
・TCP
　：コネクション型のプロトコル
　・スリーウェイハンドシェイク
　　：コネクション型プロトコルのためデータ転送を行う前にコネクションの確立を行うこと。3階のやり取りによって確立される。
　・Telnet
　　：ネットワークに接続された機器を遠隔操作するために使用するアプリケーション層プロトコル。
　・HTTP
　　：HTML文書などのリソースを取り出すことができるプロトコル。Webにおけるデータ交換の基礎。
　・FTP
　　：File Transfer Protocol。特定のPC間でファイル転送するときに使用するアプリケーション層のプロトコル。
　・SMTP
　　：Simple Mail Transfer Protocol。電子メるを送信するためのプロトコル。アプリケーション層に位置する。
・TCPヘッダ
　：非常に細かく11のフィールドから構成される
・UDP
　：コネクションレス型のプロトコル
　・RTP
　　：Real-time Tranport Protocol。音声や動画のような連続するデータの流れをリアルタイムに伝送するための通信プロトコル。
　・RTCP
　　：RTP Control Protocol。RTPなどを用いてデータをリアルタイムで伝送する際に送受信制御などを行うプロトコル。
　・DHCP
　　：Dynamic Host Configuration Protocol。IPネットワークに新たに接続された機器にIPアドレスなど通信に必要な設定を自動的に割り当てるためのプロトコル。
　・TFTP
　　：Trivial File Transfer Protocol。IPネットワーク上でふぁいり転送を行うためのプロトコル。自動配布などの用で簡易なつくりになっている。
　・RIP
　　：Routing Information Protocol。ルーティングプロトコルの一つ。通信機器間で経路情報を交換しある地点から別の地点までの最短経路を割り出すのに使われる。ASのIGP。
・UDPヘッダ
　：コネクションレスのため仕手にデータを送る効率を重視しておりヘッダには相手と自分のポート番号のフィールドくらいしかない。
